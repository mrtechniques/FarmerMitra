FarmerMitra is an AI-powered agricultural tool that helps farmers **instantly detect crop diseases** from a photo of a leaf and receive **actionable, real-time remedies**. It uses a custom-trained Deep Learning model (**ConvNeXt-Tiny + CBAM Attention**) combined with **Google Gemini AI** for remedy generation, supporting **7 regional Indian languages**.

---

## ⚡ Quick Start (TL;DR)

```bash
# 1. Clone the repo
git clone https://github.com/yourusername/FarmerMitra.git && cd FarmerMitra

# 2. Start the Backend (Terminal 1)
cd backend
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env    # Then add your Gemini API key
python app.py           # Runs on http://localhost:5050

# 3. Start the Frontend (Terminal 2)
cd frontend
npm install
npm run dev             # Runs on http://localhost:3000
```

> **That's it!** Open `http://localhost:3000` in your browser and upload a leaf photo.

## 🌱 What is FarmerMitra?

FarmerMitra acts as a **digital companion for farmers**. By simply uploading a photo of a leaf, farmers receive an instant health assessment. If a disease is detected, FarmerMitra provides:

- 🔬 **Accurate diagnosis** with confidence scores
- 💊 **Customized remedies** generated by Gemini AI
- ⏱️ **Recovery timelines** for each disease
- 🛡️ **Prevention strategies** to protect future crops
- 🌐 **Multilingual support** for regional accessibility

---

## ✨ Features

| Feature | Description |
|---------|-------------|
| 🧠 **Deep Learning Detection** | Classifies diseases across Apple, Grape, and Tomato crops using ConvNeXt-Tiny + CBAM architecture |
| 💊 **Dynamic Remedies** | Google Gemini AI generates specific, actionable treatment strategies |
| 🌐 **Multilingual Support** | Hindi, Kannada, Malayalam, Tamil, Marathi, Telugu, Bengali |
| 🖼️ **Image Validation** | Rejects non-plant or blurry images using Gemini Vision |
| 📸 **Batch Analysis** | Analyze up to 15 photos at once with GPS-based risk mapping |
| 📍 **Adjacent Risk Detection** | Uses Haversine distance to warn healthy fields within 100m of infections |
| 🎨 **Modern UI** | Responsive interface built with React 19, Tailwind CSS v4, and Framer Motion |

## 🚀 Getting Started

### Prerequisites

Make sure you have these installed on your machine:

| Tool | Version | Download |
|------|---------|----------|
| **Python** | 3.10+ | [python.org](https://www.python.org/downloads/) |
| **Node.js** | v18+ | [nodejs.org](https://nodejs.org/) |
| **Git** | Any | [git-scm.com](https://git-scm.com/) |
| **Git LFS** | Any | [git-lfs.com](https://git-lfs.com/) _(for model weights)_ |

### Step 1 — Clone the Repository

```bash
git clone https://github.com/mrtechniques/FarmerMitra.git
cd FarmerMitra
git lfs pull    # Downloads the model weights (~100MB+)
```

### Step 2 — Get a Gemini API Key

You need a **free** Google Gemini API key for the AI-powered remedy generation:

1. Go to [Google AI Studio](https://aistudio.google.com/apikey)
2. Click **"Create API Key"**
3. Copy the generated key — you'll need it in the next step

### Step 3 — Set Up the Backend

Open **Terminal 1** and run:

```bash
# Navigate to the backend
cd backend

# Create and activate a virtual environment
python -m venv venv

# Activate it:
# macOS/Linux:
source venv/bin/activate
# Windows:
venv\Scripts\activate

# Install Python dependencies
pip install -r requirements.txt

# Create your environment file from the template
cp .env.example .env
```

Now open `backend/.env` in any text editor and paste your Gemini API key:

```env
GEMINI_API_KEY=paste_your_actual_key_here
GEMINI_MODEL=gemini-2.0-flash
```

Start the backend server:

```bash
python app.py
```

✅ You should see: `Starting server on http://localhost:5050`

### Step 4 — Set Up the Frontend

Open **Terminal 2** (keep the backend running!) and run:

```bash
# Navigate to the frontend
cd frontend

# Install Node.js dependencies
npm install

# Start the development server
npm run dev
```

✅ You should see: `Local: http://localhost:3000`

### Step 5 — Use the App! 🎉

Open your browser and go to **`http://localhost:3000`**. Upload a photo of a leaf and watch the magic happen!

**Request Body:**

```json

  "image": "data:image/jpeg;base64,...",
  "language": "en"
}
```

_Supported languages: `en`, `hi`, `kn`, `ml`, `ta`, `mr`, `te`, `bn`_

**Response (200 OK):**

```json
{

  "leafType": "Tomato",
  "confidence": 98.4,
  "remedies": [
    "Apply fungicides containing chlorothalonil.",
    "Remove lower infected leaves."
  ],
  "recoveryTime": "2-3 weeks with treatment",
  "details": "Early blight is caused by Alternaria solani...",
  "rawClass": "Tomato___Early_blight",
  "top3": [
    {
      "class": "Tomato___Early_blight",
      "plant": "Tomato",
      "disease": "Early Blight",
      "confidence": 98.4
    }
  ]
}
```

</details>

<details>
<summary><strong>POST /api/batch-predict</strong> — Batch analysis with GPS risk mapping</summary>

Processes up to 15 images concurrently and computes adjacent risk via Haversine distance.

**Request Body:**

```json
{
  "photos": [
    {
      "image": "data:image/jpeg;base64,...",
      "lat": 34.0522,
      "lng": -118.2437,
      "label": "Sector A"
    }
  ],
  "language": "en"
}
```

**Response (200 OK):**

```json
{
  "results": [
    {
      "label": "Sector A",
      "disease": "Healthy",
      "leafType": "Grape",
      "confidence": 99.1,
      "isHealthy": true,
      "severity": "none",
      "adjacentRisk": true,
      "protectiveActions": ["Apply preventive fungicide spray immediately."]
    }
  ]
}
```

</details>

### Error Codes

| Code | Meaning |
|------|---------|
| **400** | Bad Request — Missing required fields (e.g., no image provided) |
| **422** | Unprocessable Entity — Image failed semantic validation (not a plant leaf) |
| **500** | Internal Server Error — Server-side PyTorch or Gemini crash |
| **503** | Service Unavailable — ML model failed to load into memory |

## 🧪 Model Training

The disease detection model is trained using a combination of the **PlantVillage**, **PlantDoc**, and specialized leaf datasets.

| Detail | Value |
|--------|-------|
| **Notebook** | `FarmerMitra_Training_Pipeline.ipynb` |
| **Architecture** | ConvNeXt-Tiny + CBAM (Convolutional Block Attention Module) |
| **Environment** | Google Colab (GPU recommended) |
| **Datasets** | Auto-downloaded via `kagglehub` |

### How to Retrain

1. Open `FarmerMitra_Training_Pipeline.ipynb` in [Google Colab](https://colab.research.google.com/)
2. The notebook handles everything: dataset download, augmentation, training, and evaluation
3. After training, download the `.pth` weights file and place it in `backend/models/`

> 💡 **Community**: Feel free to download, modify, and share the notebook. It's designed to be plug-and-play!

## 🌍 Deployment

### Production Build

**Frontend:**

```bash
cd frontend
npm run build
```

This generates an optimized `dist/` folder with static assets (HTML/CSS/JS), ready for deployment.

**Backend:**

Use a production WSGI server instead of Flask's built-in server:

```bash
cd backend
pip install gunicorn
gunicorn app:app --workers 4 --bind 0.0.0.0:5050
```

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `GEMINI_API_KEY` | ✅ Yes | Your Google Gemini API key |
| `GEMINI_MODEL` | Optional | Model to use (default: `gemini-2.0-flash`) |
| `PORT` | Optional | Backend server port (default: `5050`) |

### Recommended Hosting

| Layer | Platforms |
|-------|-----------|
| **Frontend** | Vercel, Netlify, AWS Amplify |
| **Backend** | Render, Railway, AWS EC2, DigitalOcean _(requires `torch` — needs ≥4GB RAM)_ |

## 🔧 Troubleshooting

<details>
<summary><strong>❌ "Model not loaded. Check server logs." (Error 503)</strong></summary>

**Cause:** The `.pth` weight file is missing from `backend/models/`.

**Fix:**
1. Run `git lfs pull` to download the model weights
2. Verify `backend/models/farmer-mitra_weights.pth` exists
3. Restart the backend server

</details>

<details>
<summary><strong>❌ Port 5050 is already in use</strong></summary>

**Cause:** Another process is using port 5050.

**Fix:**
```bash
PORT=5051 python app.py
```
Then update the proxy target in `frontend/vite.config.ts` to match the new port.

</details>

<details>
<summary><strong>❌ "AI Validation Service is temporarily unavailable"</strong></summary>

**Cause:** Invalid, expired, or rate-limited Gemini API key.

**Fix:**
1. Check your key at [Google AI Studio](https://aistudio.google.com/apikey)
2. Ensure `backend/.env` has the correct `GEMINI_API_KEY`
3. Restart the backend server

</details>

<details>
<summary><strong>❌ Out of Memory (OOM) crash on backend</strong></summary>

**Cause:** PyTorch is allocating too much memory, especially during batch predictions.

**Fix:**
- Reduce the number of images in batch mode
- Run on a machine with ≥4GB RAM
- Close other memory-heavy applications

</details>
